# File Tree Design

## Overview

Design for the file tree component in the sidebar.

## Requirements

1. Display directory structure in a tree format
2. Expand/collapse directories on click
3. Lazy loading for performance (load children on expand)
4. Show file icons based on file type
5. Click file to open in editor

## Data Structure

### FileEntry (Rust)

```rust
#[derive(Debug, Clone, Serialize)]
pub struct FileEntry {
    pub name: String,
    pub path: String,
    pub is_dir: bool,
    pub is_hidden: bool,
    pub children: Option<Vec<FileEntry>>,
}
```

### Tauri Commands

```rust
#[tauri::command]
fn read_directory(path: String) -> Result<Vec<FileEntry>, String>

#[tauri::command]
fn get_home_directory() -> Result<String, String>
```

## Frontend Components

### FileTree.svelte

Main container component that manages the tree state.

```
Props:
- rootPath: string (root directory to display)

State:
- expandedPaths: Set<string> (currently expanded directories)
- selectedPath: string | null (currently selected file/folder)
```

### FileTreeItem.svelte

Individual tree item (file or folder).

```
Props:
- entry: FileEntry
- depth: number (indentation level)
- expanded: boolean
- selected: boolean

Events:
- on:toggle (for folders)
- on:select (for files)
```

## Styling

- Indent: 16px per level
- Item height: 24px
- Icon size: 16px
- Colors: Use CSS variables from theme

## Exclude Patterns

Default exclusions:
- node_modules
- .git
- .DS_Store
- target (Rust)

## Implementation Steps

1. Create Rust commands for file system access
2. Create FileTreeItem.svelte component
3. Create FileTree.svelte component
4. Create fileTreeStore for state management
5. Integrate with Sidebar component
6. Add file type icons

## Acceptance Criteria

- [ ] Directory tree displays in sidebar
- [ ] Folders can be expanded/collapsed
- [ ] Files can be selected
- [ ] Hidden files/folders are handled
- [ ] node_modules and .git are excluded by default
- [ ] Performance is acceptable for large directories
