# Terminal Design

## Overview

Design for the terminal component using xterm.js.

## Requirements

1. Display terminal emulator in main content area
2. Support shell interaction (bash/zsh)
3. Handle input/output streaming
4. Support ANSI colors and formatting
5. Resize handling

## Architecture

### Backend (Rust)

Uses Tauri's shell plugin or PTY (pseudo-terminal) for shell interaction.

```rust
// PTY-based terminal
- Spawn shell process (bash/zsh)
- Stream stdin/stdout between frontend and shell
- Handle resize events
```

### Dependencies

**Rust:**

- tauri-plugin-shell (for basic shell commands)
- portable-pty or pty-process (for full PTY support)

**Frontend:**

- xterm (terminal emulator)
- xterm-addon-fit (auto-resize)
- xterm-addon-web-links (clickable links)

## Components

### Terminal.svelte

Main terminal component that wraps xterm.js.

```
State:
- terminal: Terminal instance
- ptyProcess: PTY process reference
- connected: boolean
```

### TerminalManager (Rust)

Manages PTY processes and communication.

```rust
struct TerminalManager {
    processes: HashMap<u32, PtyProcess>,
    next_id: u32,
}
```

## Tauri Commands

```rust
#[tauri::command]
async fn create_terminal(cwd: Option<String>) -> Result<u32, String>

#[tauri::command]
async fn write_terminal(id: u32, data: String) -> Result<(), String>

#[tauri::command]
async fn resize_terminal(id: u32, cols: u16, rows: u16) -> Result<(), String>

#[tauri::command]
async fn close_terminal(id: u32) -> Result<(), String>
```

## Events

```rust
// Tauri events for terminal output
app.emit("terminal-output", TerminalOutput { id, data })
```

## Styling

- Font: JetBrains Mono, Menlo, monospace
- Font size: 14px
- Background: var(--bg-primary)
- Colors: Match VS Code dark theme

## Implementation Steps

1. Install xterm.js dependencies
2. Create Terminal.svelte component
3. Add Rust terminal commands (PTY)
4. Integrate with MainContent
5. Add resize handling

## Simplified v0.0.1 Approach

For v0.0.1, use a simpler approach:

- Use Tauri shell plugin for command execution
- Basic command input/output display
- Full PTY support in future version

## Acceptance Criteria

- [ ] Terminal displays in main content
- [ ] Can execute shell commands
- [ ] Output is displayed correctly
- [ ] ANSI colors are rendered
- [ ] Terminal resizes with window
