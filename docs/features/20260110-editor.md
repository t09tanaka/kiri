# Editor Design

## Overview

Design for the code editor component using CodeMirror 6.

## Requirements

1. Display and edit text files
2. Syntax highlighting for common languages
3. Line numbers
4. Read file content from backend
5. Save file changes

## Architecture

### Backend (Rust)

```rust
#[tauri::command]
fn read_file(path: String) -> Result<String, String>

#[tauri::command]
fn write_file(path: String, content: String) -> Result<(), String>
```

### Frontend

Using CodeMirror 6 with modular architecture.

## Dependencies

**Frontend:**

- @codemirror/state
- @codemirror/view
- @codemirror/commands
- @codemirror/language
- @codemirror/lang-javascript (JS/TS)
- @codemirror/lang-rust

## Components

### Editor.svelte

Main editor component wrapping CodeMirror.

```
Props:
```

## Language Detection

Detect language from file extension:

- .ts, .tsx -> TypeScript/JavaScript
- .js, .jsx -> JavaScript
- .rs -> Rust
- .json -> JSON
- .md -> Markdown
- .css, .scss -> CSS
- .html, .svelte -> HTML
- .toml -> TOML (fallback to plain)

## Styling

- Theme: One Dark (matches VS Code)
- Font: JetBrains Mono, Menlo, monospace
- Font size: 14px

## Implementation Steps

1. Install CodeMirror dependencies
2. Create file read/write Rust commands
3. Create Editor.svelte component
4. Add language detection
5. Integrate with MainContent
6. Add save functionality (Cmd+S)
