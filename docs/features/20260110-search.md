# Search Functionality Design

## Overview

Provide file and content search capabilities with a quick-open dialog.

## Requirements

1. Quick Open dialog (Cmd+P) for file search
2. Content search (Cmd+Shift+F) for text in files
3. Fast, incremental search with fuzzy matching

## Architecture

### Backend (Rust)

```rust
#[derive(Debug, Clone, Serialize)]
pub struct SearchResult {
    pub path: String,
    pub name: String,
    pub is_dir: bool,
    pub matches: Option<Vec<SearchMatch>>,
}

#[derive(Debug, Clone, Serialize)]
pub struct SearchMatch {
    pub line: usize,
    pub content: String,
    pub start: usize,
    pub end: usize,
}

#[tauri::command]
fn search_files(
    root_path: String,
    query: String,
    max_results: usize
) -> Result<Vec<SearchResult>, String>

#[tauri::command]
fn search_content(
    root_path: String,
    query: String,
    max_results: usize
) -> Result<Vec<SearchResult>, String>
```

### Frontend Components

1. **QuickOpen.svelte** - Modal dialog for file search
2. **SearchPanel.svelte** - Side panel for content search

## UI Design

### Quick Open (Cmd+P)

- Modal dialog centered on screen
- Search input at top
- File list with fuzzy match highlighting
- Keyboard navigation (Up/Down, Enter)

### Content Search (Cmd+Shift+F)

- Panel in sidebar
- Search input with regex toggle
- Results grouped by file
- Click to open file at line

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| Cmd+P | Open Quick Open |
| Cmd+Shift+F | Open Content Search |
| Escape | Close dialog/panel |
| Enter | Open selected file |
| Up/Down | Navigate results |
