# Contributor docs

Canonical, human-facing reference material for working on kiri. If you arrived
here from the root [`CONTRIBUTING.md`](../../CONTRIBUTING.md), this directory
holds the longer-form material that didn't fit there.

| File | What it covers |
|---|---|
| [`testing.md`](testing.md) | Test categories, coverage goals, and how to choose between unit / browser / integration / e2e |

Anything inside `.claude/` in this repo is intentionally local-only (it is in
`.gitignore`). Those notes are written for AI coding agents and may be in
Japanese; nothing in them should be a prerequisite for a human contributor.
If you find policy in an agent-only note that humans also need to know, please
move it into this directory and link from `CONTRIBUTING.md`.
